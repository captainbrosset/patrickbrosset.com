First, let's talk about naming here. -> Lanes This might be new to some of you. Masonry is the way that most people have come to know about this type of layout. It's still the most common name for it. But, it actually comes from a popular JS library by David DeSandro. You can use this lib in browsers today, and it's been very popular for a long time. Mozilla was the first to provide an experimental implementation of this layout, right after they implemented CSS grid, so almost 10 years ago now. When they did, they preserved the name "masonry" But now, in a crazy turn of events, the latest spec, and current implementations, don't use that term anymore. The new name grid-lanes. Think of it as swim lanes, or maybe traffic lanes. And the reason for this is that masonry is 1D, you only define layout along one axis.
