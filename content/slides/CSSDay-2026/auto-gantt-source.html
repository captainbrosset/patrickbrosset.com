.gantt {
  display: grid-lanes;
  grid-template-rows: [timeline] .5rem [frontend] 4rem [backend] 4rem [devops] 4rem;
}

[data-workstreams="frontend"] { grid-row: frontend; }
[data-workstreams="backend"]  { grid-row: backend; }
[data-workstreams="devops"]   { grid-row: devops; }
[data-workstreams="all"]      { grid-row: 2 / -1; }

.task {
  // Width comes straight from the task duration in days.
  inline-size: calc(attr(data-length type(<number>)) * var(--days-to-width));
}
// Interactive resize: update data-length while dragging.
task.dataset.length = Math.max(1, startLength + daysDelta);
Interesting bit here is grid-template-rows, which is enough of a clue for the engine to know this is a row orientation grid-lanes and the named lines. This way, I can assign each work item to the right team, based on an HTML attribute. Another random fun bit here is the inline size of the tasks. Each task has a data-length attribute, a number of days, And I'm using the power of the typed attr() function to turn this into a CSS length, by multiplying with a ratio. This also means that I can resize in JS by just changing the data-length attribute.
